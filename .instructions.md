# THINK++ Agent Instructions

You are working on **THINK++** — an interactive educational platform for CS fundamentals. The codebase is a Next.js 16 app with Python backend, Tailwind 4, and a filesystem-based lesson registry.

## Critical Rules (ALWAYS follow)

### Next.js 16 Breaking Changes
- **Read** `node_modules/next/dist/docs/` before writing Next.js code
- Do NOT use deprecated APIs from earlier Next.js versions
- Heed all deprecation warnings in build output

### Content System (Non-Negotiable)
- **NEVER hardcode lesson routes** — use the filesystem-based registry
- **ALWAYS** update `lib/content-registry.ts` when:
  - Adding/renaming lessons
  - Adding/renaming subjects
  - Changing `LESSON_ORDER` or `TOPIC_LABELS`
  - Modifying `SUBJECT_META` for new subjects
- Lesson files: `content/[subject]/[topic]/[lesson].tsx`
- Update registry BEFORE testing routes

### Design System
- **Theme**: High-contrast black-and-white (CSS variables, not Tailwind colors)
- **Color Vars**: Use `var(--bg-primary)`, `var(--text-primary)`, `var(--text-secondary)`, etc.
- **Never**: `bg-blue-500`, `text-red-300`, or hardcoded hex colors
- **Spacing**: High-density lessons use `px-12 py-24` (standard)
- **Spotlight Effect**: Global; never re-implement per-component
- **Design Tokens**: Located in `app/styles/` with `@theme` mapping in `theme.css`

### Performance
- Use `SimulationSkeleton` for loading states
- Keep lesson components focused (simulations = client-side, data = server-side)
- Optimize heavy visualizations with React.memo or dynamic imports

## File Structure Conventions

```
content/
  [subject]/           # ai-ml, algorithms, etc.
    [topic]/          # classification, sorting, etc.
      [lesson].tsx    # Individual lesson components
lib/
  content-registry.ts  # Single source of truth (DO NOT skip this)
  components/        # Reusable components (not subject-specific)
app/
  styles/            # Tailwind 4 + CSS variables
  page.tsx           # Homepage
  layout.tsx         # Root layout
```

## Development Workflow

### Adding a New Lesson
1. Create `content/[subject]/[topic]/[lesson].tsx`
2. Update `lib/content-registry.ts`:
   - Add to `LESSON_ORDER`
   - Add to `TOPIC_LABELS` if new topic
3. Add to `SUBJECT_META` if new subject
4. Run `npm run generate:lesson-loaders` (auto on dev/build/start)
5. Test route at `/learn/[subject]/[topic]/[lesson]`

### Adding a New Subject
1. Create `content/[subject]/` directory
2. Add to `SUBJECT_META` in `lib/content-registry.ts` with:
   - Icon (from lucide-react or custom)
   - Description (brief subject tagline)
3. Create first topic/lesson inside
4. Run `npm run generate:lesson-loaders`

### Styling Lessons
- Import design tokens or define locals within lesson
- Use CSS variables for all colors
- Leverage Tailwind utilities with `@theme` mappings
- High-density content = `px-12 py-24` baseline

### Backend (Python/FastAPI)
- Located in `backend/`
- Run separately: `npm run dev:backend`
- Accessible from frontend via API calls
- Use `npm run dev:all` for concurrent frontend + backend

| `npm run dev:backend` | Start Python backend (uvicorn) |
| `npm run dev:all` | Concurrent frontend + backend |
| `npm run build` | Production build |
| `npm run lint` | ESLint check |

## Optimization Tips

1. **Lesson Loading**: Use `SimulationSkeleton` instead of generic spinners
2. **Code Splitting**: Dynamic imports for heavy visualizations
3. **Styling**: Define tokens once in `app/styles/`, reuse everywhere
4. **Testing**: Run `npm run lint` before pushing
5. **Build**: Always run `npm run build` locally to catch Next.js errors

## Common Pitfalls

- ❌ Hardcoding `/learn/ai-ml/regression/linear` routes
- ❌ Using Tailwind color utilities instead of CSS variables
- ❌ Forgetting to update `content-registry.ts` after adding lessons
- ❌ Re-implementing Spotlight effect in components
- ❌ Not running `generate:lesson-loaders` after structural changes
- ❌ Ignoring Next.js 16 deprecation warnings
